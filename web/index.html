About

Sono is a high-level object-oriented and procedural scripting language developed with linguistic capabilities in mind. For that it supports multiple operators revolving around phonological analysis based upon distinctive features.

The language is capable of functioning moreover as a general scripting language, with a growing number of libraries and the capacity to interface with Java files. The language is small right now and has very little following, if any at all, but it is a language made to fill a niche that has not yet been assessed. Every day, the gap between the sciences and programming becomes smaller and smaller, and the sadly neglected science of Linguistics is equally deserving of its time in the world of programming. If not by this language does it become widepsread, than at least may it be that this language inspire another to take up that mantle.

Installation

There are no binary releases at the moment, but one can compile the source directly from the Github repo: 

$ git clone https://github.com/Nallantli/Sono
$ cd Sono

For Windows:

$ ./build-jar
$ ./build-lib

For OSX/Linux:

$ sh build-jar.sh
$ sh build-lib.sh

These commands will place all necessary files into their locations in /bin, which can be then added to the system path.

Once the system path is updated, all one needs to do is offer the interpreter a dataset consisting of base segmental values. The repo includes a slightly adjusted version of Bruce Hayes' feature chart under assets/:

$ sono -d assets/hayes.tsv

This will generate the cache, stored in your home directory under .sono/. Now you can entire the console or run a file!

$ sono
Sono {VERSION}
Phonological Data Loaded From <{DATA}>
Loaded System Library
>
$ sono examples/hello.so
Hello World!

Hello World

Below is the classic "hello, world!" script; with one line the task is done:

println("Hello World!");

Phonology

There are multiple examples to be seen in the repo, but a few minor demonstrations can never be so bad. 

var rule = S |> [-|syl, +|cons, +|voice, -|nasal] -> [-|voice] // null .. "#";

That value above is a Rule, a phonological transformation rule. When applied to a Word, it transforms that value accordingly. The rule above is a final-consonant devoicing rule, so any consonant at the end of a Word that is not a nasal will become [-voice].

println(`taːg` >> rule);
println(`taːgə` >> rule);

Output:

taːk
taːgə

Note that the second Word is unaffected - it does not end with a consonant. Rules can be much more powerful and versatile using a variety of tokens and parameters further explained in the repo (but soon to be transported here).
